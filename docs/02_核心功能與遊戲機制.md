# 二、 核心功能與遊戲機制 (Core Mechanics)

## 2.1 玩家帳號與安全認證 (User Authentication)
*   **帳號註冊**：玩家必須在首次進入網頁時註冊帳號，輸入用戶名、電子信箱、密碼（密碼需經過後端 Hash 加密儲存）。
*   **登入與 Token 驗證**：使用 JSON Web Token (JWT) 進行身分驗證。玩家登入後，JWT Token 將安全地儲存在瀏覽器的 `SessionStorage` 中，後續所有涉及數值變更的 API 請求都必須攜帶此 Token。

## 2.2 50 隻怪獸倉庫容量 (50-Monster Roster Limit)
每個帳號最多可同時擁有 50 隻活體怪獸（包含孵化中與孵化完成的怪獸）。
*   **怪獸管理**：隨時在倉庫列表中查閱這 50 隻怪獸的健康狀態。
*   **鎖定功能**：對特定優秀基因的怪獸進行「解鎖/鎖定」，防止誤操作進行放生或不慎提取為晶片。

## 2.3 生命週期、飢餓與疾病致死機制 (Life Cycle & Mortality)
本系統移除了傳統的「老死/壽終正寢」概念。怪獸在正常照料下可達到「永生」狀態，但引進了嚴格的「疏忽致死」機制。

### 2.3.1 狀態屬性與遞減規則
*   **飽食度 ($Hunger$)**：範圍為 $0$ 至 $100$。為配合成年玩家作息，飽食度從 $100$ 降至 $0$ 固定需要 **24 小時**（約每小時遞減 $4.16$ 點），每天僅需登入餵食一次即可維持生命。
*   **清潔度 ($Cleanliness$)**：範圍為 $0$ 至 $100$。怪獸會定時產生排泄物，每次產生排泄物使清潔度降低 $20$ 點。
*   **疾病機制 ($Sickness$)**：生病為隨機突發事件，任何時候都可能發生。但為避免過度打擾玩家，設有 **48 小時的冷卻期**（即每次生病痊癒後，48 小時內絕對不會再次生病）。生病時戰鬥屬性瞬間下降 $70\%$。
*   **睡眠機制 (Sleep)**：怪獸在晚上 22:00 至早上 06:00 會進入自然睡眠時間。此外，也可透過「💡 開關燈」按鈕強制使怪獸進入睡眠。睡眠期間會停止走動並顯示 Zzz 動畫。

### 2.3.2 培育失誤判定 (Care Mistakes)
「失誤次數」是決定怪獸進化屬性分歧（疫苗、資料、病毒）的核心指標。每發生以下一種情況，後端將會把該怪獸的 `mistake_count` 增加 1 次：
1.  **飢餓放任 (Hunger Neglect)**：飽食度降為 $0$（觸發飢餓警告）後，超過 **2 小時** 仍未餵食。
2.  **排泄物放任 (Poop Neglect)**：清潔度低於 $30$，或畫面上有排泄物超過 **2 小時** 未清理；或是怪獸在**有排泄物的狀態下進入睡眠**。
3.  **生病放任 (Sickness Neglect)**：進入「生病」狀態後，超過 **4 小時** 未給予藥物治療。
4.  **睡眠干擾 (Sleep Neglect)**：到達睡眠時間後，超過 **2 小時** 未關燈讓怪獸入睡。

### 2.3.3 防退坑保護：時光靜止倉機制 (Time-Freeze Pod)
為避免成年玩家因忙碌導致怪獸意外死亡而產生巨大挫折感，本系統沒有真正的「疏忽死亡」，而是引入「時光靜止倉」作為最後保護網。
由於系統採用 Serverless 架構，伺服器透過「懶加載計算 (Lazy Evaluation)」來判定生命狀態：
*   **觸發靜止條件**：
    *   **飢餓瀕死**：當飽食度降為 $0$ 且持續未餵食達 **23 小時**（距離餓死僅剩 1 小時）時。
    *   **疾病瀕死**：生病未治療達 **47 小時**（距離病死僅剩 1 小時）時。
    只要滿足上述任一條件，系統會強制介入，將怪獸收入「時光靜止倉」，此時怪獸的一切狀態與時間流逝都會被凍結。
*   **解凍與代價**：玩家上線後會發現怪獸處於靜止倉中。玩家必須花費 **100 金幣** 將怪獸解凍。解凍後的怪獸仍處於瀕死邊緣，玩家必須立即餵食或餵藥。
*   **真正的死亡**：怪獸不會因飢餓或疾病自然死亡，只有當玩家主動選擇「放生」或「轉化為資料晶片」時，怪獸才會從活體倉庫中消失。

### 2.3.4 老化衰退機制 (Aging Decline but Immortal)
*   **巔峰期**：進化為「究極體」後，享有 30 天實時時間的屬性全盛期。
*   **衰退期**：達到第 31 天起，實際戰鬥值每天扣減 $1\%$。保底不低於幼年期初始值。
*   **基因保值**：基因基礎值終身不衰退，可作為繁衍親代。

## 2.4 道具功能與基因晶片鑲嵌機制 (Items & Chips)
*   **消耗性道具**：包含各種回復飽食度的食物、治病藥物、提升突變機率的繁衍催化劑。
*   **晶片插槽 (Chip Slots)**：每隻怪獸擁有 2 個晶片槽，分別於成熟期與完全體解鎖。可鑲嵌基因晶片獲得額外屬性加成。

## 2.5 經濟系統與免費收入來源 (Economy & Free Income)
*   每日簽到 (Daily Login)
*   每日任務 (Daily Quests)
*   戰鬥結算獎勵 (Combat Rewards)
*   公會津貼 (Guild Salary)
*   放生獎勵 (Release Reward)

## 2.6 怪獸圖鑑系統 (Pokedex)
為幫助玩家規劃培育路線，系統內建「怪獸圖鑑」功能：
*   **圖鑑總覽**：玩家可隨時查閱遊戲內所有可能的怪獸分支、像素圖案與屬性。
*   **進化提示**：圖鑑內會明確標示每隻怪獸達到該階段所需的「進化條件」（例如：需 50 戰與 70% 勝率、訓練次數等），作為玩家養成的目標指引。

## 2.7 測試與時間縮放機制 (Debug Mode)
為便於功能驗證與平衡性調整，系統提供開發調試設定：
*   **時間縮放因子 (`debug_time_multiplier`)**：後端環境變數支援調整時間流速，預設為 $1$。調試環境下可設置為 $60$（1分鐘等於1小時）或 $3600$（1秒鐘等於1小時），加速驗證生命狀態遞減與餓死病逝邏輯。
*   **開發者主控台**：僅供持有 `is_admin` 權限之帳戶在本地/測試伺服器訪問，可一鍵執行「清空飢餓值」、「引發疾病」、「前進 1 天」、「重置進化屬性」等操作。

## 2.8 社交與通訊功能 (Social & Communication)
為保持競技場 (Arena) 專注於戰鬥體驗，所有交友與通訊功能皆整合至「社交 (Social)」模組：
*   **世界聊天 (World Chat)**：全伺服器互通的即時頻道，方便玩家交流培育心得，所有對話永久留存以防範不法。
*   **大廳與好友**：玩家可於大廳查看目前在線玩家的怪獸陣容並發起挑戰；也可透過 UUID 互加好友，進行每日體力贈送互動。
