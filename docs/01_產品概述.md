# 一、 產品概述 (Product Vision)

## 1.1 產品定位
一款融合經典電子雞情懷與現代多人在線互動的網頁養成 RPG。玩家透過註冊帳號，可同時在線培育、儲存多達 50 隻怪獸，並與全伺服器玩家進行即時聊天、公會合作、好友切磋、非同步 PvP 競技場以及共同討伐世界王。

## 1.2 技術架構藍圖 (Serverless Architecture)
為實現 GitHub Pages 靜態部署且具備多人互動，系統採用「Serverless (無伺服器)」架構：

*   **前端 (Client)**：託管於 GitHub Pages (HTML5 / Single Page Application / Tailwind CSS / React)，負責 UI 渲染、本地快取與狀態推算。
*   **前端快取管理 (Cache Invalidation)**：`index.html` 必須設定不快取 (`Cache-Control: no-cache`)，配合 Vite 建置時會自動對 JS/CSS 檔案名稱產生 Hash 碼，確保使用者每次開啟網頁時都能自動獲取最新版本，無須手動清除快取 (`Ctrl+F5`)。
*   **後端與資料庫 (Backend & Database)**：完全採用 **Firebase (Firebase Auth + Cloud Firestore)**。負責處理用戶註冊與登入驗證、即時資料同步 (onSnapshot)、以及資料庫存儲。

> [!NOTE]
> **💡 系統架構具體技術選型**
> *   **前端技術**：`React` + `Vite` + `Tailwind CSS` + `Lucide Icons`。
> *   **後端與資料庫**：`Firebase Auth` (安全認證) + `Cloud Firestore` (NoSQL 即時資料庫)。
> *   **視覺與音效**：採用經典電子雞（LCD螢幕）風格，外層背景為簡潔白色，而遊戲機台與 LCD 螢幕維持深色模式以突顯暗黑科技感。LCD顯示比例為寬版長方形（400x250左右視覺尺寸），以供怪獸較長距離的走動。怪獸點陣圖以高亮青色 (Cyan) 呈現，並支援睡眠關燈模式，搭配 8-bit 電子音效擴充性。

## 1.3 使用者故事與交互設計 (User Stories)

*   **帳號安全性故事**：
    > **身為** 新玩家，**我想要** 能直接在行動網頁上註冊帳號並用密碼登入，**以便於** 我的怪獸資料可以安全保存在伺服器，且在不同手機/電腦瀏覽器上登入都能同步進度。
*   **多隻培育與防呆故事**：
    > **身為** 怪獸搜集狂玩家，**我想要** 同時飼養多達 50 隻怪獸，**以便於** 我可以分類出「實戰主力隊」，「種馬繁衍隊」與「待培育幼苗」，並將主力怪獸進行「鎖定」，防止誤操作進行放生或不慎提取為晶片。
*   **責任養成與生病故事**：
    > **身為** 核心養成玩家，**我想要** 在怪獸飽食度歸零或生病時收到明顯的警示 UI（如紅色骷髏頭或飢餓倒數計時），**以便於** 我能即時在 24 小時（飢餓）與 48 小時（疾病）的時限內給予餵食或投藥，防止愛寵病逝或餓死。
*   **戰力公平競爭故事**：
    > **身為** 數值考究型玩家，**我想要** 我的戰鬥勝率完全基於我悉心培養的屬性差，**以便於** 在 PvP 競技場或世界王共鬥中，弱者無法憑藉純隨機的運氣推翻我花費大把精力與資源堆疊出的數值優勢（僅容許 $\pm 10\%$ 的隨機誤差）。
