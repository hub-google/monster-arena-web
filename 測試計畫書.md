# 怪獸對打機網頁版 - 系統測試計畫書 (Test Plan)

本測試計畫書模擬了一名新玩家從首次載入網頁、註冊帳號開始，到體驗遊戲內所有核心養成與戰鬥系統的完整使用者旅程 (User Journey)。

## 🎯 測試執行規範 (Testing Methodology)
根據要求，所有的 E2E 自動化測試都必須嚴格遵守以下規範：
1. **線上環境測試 (GitPages)**：所有測試必須直接對準部署在線上的 GitHub Pages 正式環境（即 `https://hub-google.github.io/monster-arena-web/`）進行驗證，以確保最終上線版本的穩定性，而非僅使用本地端 (localhost) 進行測試。
2. **截圖驗證機制 (Screenshots Validation)**：為確保測試具備可追溯性與真實性，**每一個測試情境與關鍵操作步驟，測試腳本都必須拍攝並儲存螢幕截圖**。測試完成後必須提供所有的截圖供使用者檢視，作為測試確實有被執行且畫面符合預期的視覺證據。
